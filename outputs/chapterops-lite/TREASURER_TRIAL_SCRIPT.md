# ChapterOps Lite — Treasurer Trial Script

Use this as a loose script when walking through the product. Keep it conversational.

## Opening

“I built this as a simple chapter operations system. The idea is to stop using separate spreadsheets, group chat reminders, and manual status updates. Everything important lives in one Google Sheet, and the weekly executive report is generated automatically.”

## Treasurer-specific pitch

“For the treasurer, the big value is that dues and reimbursements are connected to the rest of chapter operations. You can see total owed, collected, outstanding balance, overdue balances, pending reimbursements, and approved unpaid reimbursements without rebuilding a report every week.”

## Demo path

1. Start on Dashboard.
2. Show total active members.
3. Show dues owed, collected, outstanding balance, and overdue count.
4. Open Dues and explain each row.
5. Open Reimbursements and explain approval and paid status.
6. Open Events and Attendance.
7. Run **ChapterOps → Refresh Dashboard**.
8. Run **ChapterOps → Generate Weekly Exec Report**.
9. Show the report sections.
10. Explain that **Email Exec Report** sends the same summary to the executive email list.

## Suggested language for dues

“The Dues tab is the treasurer’s weekly working table. The only things you normally update are amount paid, payment status, payment plan, and notes. Balance updates from amount owed minus amount paid.”

## Suggested language for reimbursements

“Reimbursements are tracked separately so requests do not get lost in texts. A request can be pending, approved, or denied, and approved requests can still be unpaid until the treasurer marks them paid.”

## Suggested language for attendance

“Attendance here is operational recordkeeping. It helps officers know who may need communication or support. It is not for public comparison or member-vs-member scoring.”

## Questions to ask the treasurer

- Would this replace your current dues spreadsheet?
- Are any dues columns missing?
- What payment statuses do you actually use?
- Would reimbursement tracking save you time?
- What would need to be protected or hidden before wider use?
- Would you want a separate form for reimbursement requests?

## If something breaks during the demo

Say:

“That’s exactly why we’re trial-running it before using it broadly. The core data model is here; we’re checking workflow friction now.”

Then write down:

- What you clicked
- What you expected
- What happened
- Whether it blocks the weekend pilot
